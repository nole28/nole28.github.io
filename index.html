// 02 — Case Study

Espres case study.

Espres is the project I use to demonstrate my current iOS level. It started as a social media-style SwiftUI app and grew into a portfolio-ready v1 with authentication, media upload, profiles, comments, search, following, saved content, and messaging.

Problem

I wanted to build something closer to a real product than a tutorial app. The goal was to create a social app where users can sign in, create content, upload media, open profiles, follow users, comment, save content, search, and message.

My role

I designed and built the app as a solo project: SwiftUI screens, Firebase data modeling, app state, uploads, navigation, profile logic, debugging, file organization, and final presentation cleanup.

Architecture

I organized the project around clear responsibilities: Models for data, Services for Firebase work, ViewModels for screen state, Views for feature screens, and Shared components for reusable UI.

  • SwiftUI views for screens and reusable components
  • ObservableObject services for app state and Firebase operations
  • Firestore documents for users, posts, videos, comments, followers, and chats
  • Firebase Storage for uploaded image and video files

Hardest bugs solved

The biggest work was not just adding screens. It was making the app behave consistently after real data changed.

  • Fixed follower/following count mismatch and old self-follow records
  • Cleaned video creator display so old videos no longer showed generic Creator labels
  • Fixed Explore grid clipping and post detail alignment
  • Removed developer-only UI before portfolio presentation
  • Organized Xcode files and cleaned accidental duplicate file references

What I learned

Espres taught me how much work exists between “the screen appears” and “the app is ready to show.” I learned how to connect SwiftUI state with Firebase reads and writes, handle uploaded media, debug runtime problems, keep models consistent, structure a larger project, and make decisions that improve maintainability instead of only visual appearance.

Espres

Espres is a SwiftUI social app built around real product flows. Users can authenticate, set up profiles, create posts, upload video, browse Explore, open video details, comment, save content, follow profiles, view follower/following lists, and use direct messaging.

The project uses Firebase Auth, Firestore, and Firebase Storage. The technical focus was coordinating SwiftUI state with Firebase-backed data, handling media upload and playback, keeping profile relationship data consistent, and organizing the app into a cleaner feature-based project structure.

Project structure

I organized Espres by responsibility so the project is easier to read and present. Models hold app data, Services handle Firebase and app state, ViewModels manage screen state, Views are grouped by feature, and Shared contains reusable components.

This structure is simple enough for a junior project, but clean enough for someone reviewing the code to understand where features live.
