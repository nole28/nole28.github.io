● In active development

Espres App

Espres is a SwiftUI social media app built with real app architecture instead of static UI. It includes authentication, feeds, image posts, text posts, video upload, short-form playback, comments, likes, profiles, user search, settings, account actions, view counts, and Firebase-backed data models.

The app uses Firebase Authentication, Firestore, and Firebase Storage. The hardest parts were coordinating SwiftUI state with Firebase reads and writes, keeping the MVVM structure readable, handling async media uploads, preserving temporary video files before upload, and debugging behavior that only appeared at runtime.

SwiftUI Firebase Auth Firestore Firebase Storage AVFoundation MVVM Codable Async/Await

Technical decisions

  • Used SwiftUI for the main interface because the app depends heavily on reactive state and reusable views.
  • Used Firebase Auth for account creation, login, and user identity across the app.
  • Used Firestore to structure users, posts, videos, comments, likes, and profile-related data.
  • Used Firebase Storage for uploaded images and videos to keep media separate from Firestore documents.
  • Used AVFoundation for video playback and short-form media behavior.

Swift problems solved

  • Managed SwiftUI state across feed, upload, profile, search, settings, and video screens.
  • Handled async Firebase operations while keeping the UI responsive during loading and uploads.
  • Worked with media picker file URLs and copied temporary video files before upload.
  • Debugged navigation, sheet presentation, optional data, and runtime issues in the simulator.
  • Separated feature logic into reusable views and app-specific data models.

Problems I solved

  • Modeled Firestore data for users, posts, videos, comments, likes, and profile state.
  • Handled picked video files by copying temporary PHPicker URLs before upload.
  • Debugged SwiftUI navigation and sheet presentation issues that appeared only at runtime.
  • Kept the UI responsive during Firebase uploads, reads, and loading states.

Things I learned

  • How to organize a larger SwiftUI app around tabs, reusable views, and feature-specific state.
  • How Firebase Auth, Firestore, and Storage work together in a media-heavy app.
  • How to move from preview-only UI to simulator-tested app behavior.
  • How to explain technical choices clearly in a portfolio project.
